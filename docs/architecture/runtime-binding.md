# Runtime Binding

Runtime binding is the controlled association between an orchestration decision
and the runtime mechanics used to execute it.

## Why it matters

- it separates planning from execution mechanics
- it supports capability and policy checks
- it keeps runtime choice explicit and auditable

## Ownership

- canonical routing and execution semantics: `CxRP`
- canonical runtime invocation semantics: `RxP`
- runtime binding use and enforcement: `OperationsCenter`
