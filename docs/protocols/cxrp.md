# CxRP

CxRP owns canonical cross-repo execution and routing contract semantics.

## Main objects

- `TaskProposal`
- `LaneDecision`
- `ExecutionRequest`
- `ExecutionResult`
- `RuntimeBinding`
- `CapabilitySet`

## What CxRP Is Not

- not topology ownership
- not deployment ownership
- not runtime-hosting ownership
- not orchestration implementation
